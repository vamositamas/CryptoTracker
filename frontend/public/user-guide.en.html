CryptoTracker

Complete User Guide — everything you need to track, analyse, and manage your crypto trades.

Version 1.0 · English

📋 Overview

CryptoTracker is a web-based platform designed for individual traders and teams who want a structured, data-driven view of their cryptocurrency trading activity. It records every trade, calculates profit & loss automatically using customisable formulas, and gives you rich analytics at a glance.

📊 Dashboard

KPI cards, monthly charts, token rankings, and weekday performance heatmaps.

📈 Trades

Log, edit, filter, and import trades. Net profit is calculated automatically.

🗂 Master Data

Shared reference lists for tokens, trade types, and positions used across the team.

🔢 Formulas

Customise how profit, cost, and daily metrics are calculated — no coding required.

🔍 Audit Trail

Immutable log of every create, update, and delete action across all trades.

🔐 Administration

User, group, and role management with fine-grained permission control.

🔑 Login & Registration

Logging In

  1. Open the application in your browser. You will be redirected to the login page automatically.
  2. Enter your email address and password.
  3. Click Log In. On success you are taken to the Dashboard.
⚠ Note
If login fails, double-check your email and password. Contact your administrator if you cannot access your account.

Creating an Account

  1. On the login page click "Create one" at the bottom.
  2. Enter your email, username, and a password (minimum 8 characters).
  3. The username must match your trader data folder as configured by your administrator.
  4. Click Create Account. You will be logged in automatically.

Switching Trader Profile

If your account is associated with multiple trader profiles, a profile selector will appear after login. Choose the profile you want to work with and confirm.

Logging Out

Click your username at the bottom of the sidebar, then click Log out.

🧭 Navigation

The application uses a fixed left sidebar for navigation. Click any item to go directly to that section. The active page is highlighted in blue. The bottom of the sidebar shows:

  • Language toggle — switch between English (EN) and Hungarian (HU) at any time.
  • Your username — indicates the currently signed-in user.
  • Log out — ends your session.
💡 Tip
Language preference is remembered between sessions. Switch once and the app remembers your choice.

📊 Dashboard

The Dashboard is your at-a-glance performance overview. It aggregates all recorded trades into key performance indicators (KPIs), charts, and breakdowns.

View Modes

Use the View dropdown at the top to switch between:

  • All data — aggregate statistics across your entire trade history.
  • Selected year — choose a specific year from the dropdown to focus the view.

Month-by-month breakdown of trades, net profit, profit %, average daily profit %, monthly delta %, and win rate. Use the Prev / Next buttons to paginate through years. The bar chart beneath the table visualises monthly results at a glance.

The Trades page is where you record and manage all your trading activity.

Adding a New Trade

  1. Click + New Trade (top right) or press N.
  2. Select the Token, Position, and Trade Type from the dropdowns.
  3. Enter Broker Cost, Leverage, Volume, Buy Price, Sell Price, and Close Date.
  4. Click Save Trade or press Ctrl+Enter.
⚠ Before you start
Tokens, positions, and trade types must be configured in Master Data before you can create trades. If a dropdown is empty, ask your administrator to set up the reference data.
💡 Tip
The form will warn you if leverage exceeds 100× or if the buy price appears unusually low — review these before saving.

Editing a Trade

Click the Edit icon (pencil) on any row. The row expands into an inline editor — change the values and press Save. Press Cancel to discard changes.

Deleting a Trade

Click the Delete icon (trash) on any row. A confirmation prompt will appear. Confirm to permanently remove the trade.

⚠ Warning
Deletion is permanent. The audit trail records the deletion for reference, but the trade data cannot be recovered.

Filtering Trades

Use the Filters panel (top of the trade list) to narrow results by:

  • Token — select a specific token or search by name.
  • Position — filter by position type.
  • Type — filter by trade type (Long/Short).
  • Result — show only Win or Loss trades.
  • Date range — set a From and To date.

Click Clear filters to remove all active filters at once.

Summary Bar

Above the table a summary bar shows Total Trades, Wins, Losses, and Today counts, always reflecting the currently applied filter.

Importing Trades from Excel

  1. Click the Import Excel button in the trade list header.
  2. Optionally download the template to see the required column structure.
  3. Select your Excel file. The first worksheet is read; columns must match the template layout.
  4. After a successful import a toast notification shows how many trades were added.

🗂 Master Data

Master Data contains the shared reference lists used when recording trades. Changes here are visible to the entire team immediately.

ℹ Permission required
Managing master data requires the master-data:manage permission.

Three tabs are available:

  • Tokens — cryptocurrency symbols (e.g. BTC, ETH, SOL).
  • Trade Types — direction classifiers (e.g. Long, Short).
  • Positions — market types (e.g. Spot, Futures, Margin).

Adding a Value

  1. Select the appropriate tab (Tokens, Trade Types, or Positions).
  2. Click the Add … button (e.g. "Add token").
  3. Type the value name and press Save.

Editing or Deleting a Value

Use the Edit or Delete icons on any list row. Deleting a value does not remove historical trades that reference it — it only removes it from the selection list for new trades.

🔢 Formulas

The Formulas page lets you define how trade metrics such as net profit, profit %, and daily returns are calculated. Formulas use the expr-eval expression syntax and can reference trade input fields as well as previously defined formula fields.

ℹ Permission required
Managing formulas requires the formulas:manage permission.

Built-in Variables

VariableDescription
buyPriceTrade entry price (USDT).
sellPriceTrade exit price (USDT).
volumeTrade volume (USDT).
leverageLeverage multiplier.
brokerCostBroker/exchange fee (USDT).
closeDateClose date as ISO string.

Required Formulas

Some formulas are marked Required — they cannot be deleted because other parts of the application depend on them. You can edit their expressions, but make sure the formula remains valid before saving.

Adding a Custom Formula

  1. Click Add formula.
  2. Enter a unique field name (letters, numbers, underscores — must start with a letter).
  3. Write the expression using the variables listed above or any formula name defined in a row above.
  4. Watch the live preview status — it shows whether the expression is valid before you save.
  5. Click Save formulas to apply changes to the entire team.
💡 Example
To calculate gross profit for a long trade: (sellPrice - buyPrice) * volume / buyPrice * leverage - brokerCost
⚠ Caution
Changing a required formula affects every trade calculation immediately. Test thoroughly in the preview before saving.

🔍 Audit Trail

The Audit Trail is an immutable, chronological log of every CREATE, UPDATE, and DELETE action performed on trades. It provides full accountability and change history.

ℹ Permission required
Viewing the audit trail requires the audit:read permission (typically assigned to administrators).

Audit Log Columns

ColumnDescription
TimestampExact date and time of the action.
TraderThe trader profile affected.
ActionCREATE, UPDATE, or DELETE.
Trade IDInternal identifier of the trade.
FieldWhich field was changed (for UPDATE actions).
Old ValueThe value before the change.
New ValueThe value after the change.

Filtering the Audit Log

  • Action — filter by CREATE, UPDATE, or DELETE.
  • Date range — set a From date and/or To date.

Click Clear filters to reset the view.

👤 Administration — Users

ℹ Permission required
All administration pages require the users:manage permission.

The Users page lists every registered account. For each user you can see and configure:

  • Email — login email address.
  • Username — display name and trader data folder reference.
  • Group — the group the user belongs to (determines roles and permissions).
  • Status — Active or Inactive.

Adding a User

  1. Click Add User.
  2. Fill in email, username, group, and initial password.
  3. Set status to Active for the user to be able to log in.
  4. Click Save.

Editing a User

Click Edit on any row. You can update email, username, group, status, and set a new password (leave password blank to keep the current one).

Deactivating a User

Set Status to Inactive to prevent login without deleting the account.

👥 Administration — Groups

Groups are collections of roles. Assigning a user to a group gives them all the permissions attached to the group's roles.

Creating a Group

  1. Click Add Group.
  2. Enter a Name.
  3. Select one or more Roles to attach to the group.
  4. Click Save.

Typical Group Setup

GroupSuggested Roles
Administratoradmin-full (all permissions)
Tradertrader-standard (trades:read, trades:write, dashboard:read)
Viewerviewer (dashboard:read, trades:read)

🛡 Administration — Roles

Roles hold the actual permission strings. A role can have one or more permissions, entered as a comma-separated list.

Creating a Role

  1. Click Add Role.
  2. Enter a Name.
  3. Enter Permissions as a comma-separated list (see the reference table below).
  4. Click Save.

🔒 Permissions Reference

PermissionWhat it grants
users:manageCreate, edit, and manage users, groups, and roles. Access the Administration section.
trades:readView the trade list and all trade details.
trades:writeCreate and edit trades.
trades:deleteDelete trades.
audit:readView the audit trail.
dashboard:readView the performance dashboard.
master-data:manageAdd, edit, and delete tokens, trade types, and positions.
formulas:manageEdit and save calculation formulas.
💡 Tip
A standard trader typically needs: trades:read, trades:write, trades:delete, and dashboard:read. Grant users:manage only to administrators.
